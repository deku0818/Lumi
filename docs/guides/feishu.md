# 飞书机器人使用指南

把 Lumi Agent 接到飞书，私聊机器人或在群里 @ 它即可对话——发文字、图片、文件都行。内部实现见
[`docs/architecture/feishu.md`](../architecture/feishu.md)。

---

## 一、安装

飞书 channel 用 lark-oapi 长连接。相关依赖已随主依赖默认安装，`uv sync` 即可：

```bash
uv sync
```

> 若你的网络走 SOCKS 代理（`ALL_PROXY` / `https_proxy=socks5://…`），lark 的 WebSocket 需要
> `python-socks`——已随主依赖安装，无需额外装。

## 二、飞书开放平台建机器人

1. 打开 [飞书开放平台](https://open.feishu.cn/) → **创建企业自建应用**。
2. **添加应用能力 → 机器人**。
3. **权限管理**：开通机器人所需的 IM 权限。清单以 `lumi/gateway/channels/feishu/scopes.py`
   为准——`BOT_SCOPES` 缺任一都收发不了；`OPTIONAL_SCOPES`（群信息 / 群成员 / 通讯录）不开也能用，
   只是发送者与群名退化成 `用户_xxxxxx` 兜底名。**不必照着抄**：填完凭证后用下面的接入体检自动查。
4. **事件订阅**：订阅方式选 **「使用长连接接收事件」**（无需公网回调地址），订阅事件
   `im.message.receive_v1`（接收消息）。
5. 在 **凭证与基础信息** 里拿到 **App ID** 和 **App Secret**。
6. **发布应用版本**（企业内可见即可）。权限与事件的改动**只在发布后生效**，漏这步的表现与
   完全没配一模一样：长连接照常连上，但一条消息都收不到。

> **接入体检**：桌面端「设置 → 渠道 → 飞书」填完 App ID / Secret 后，「接入体检」会一次查出
> 凭证、权限、事件订阅、版本发布四项各自的状态（数据取自开放平台「应用版本信息」接口）。
> 缺权限或缺事件时直接给预填好的开放平台直达链接，点过去确认即可，不用自己对照清单勾。
>
> 首次体检大概率会先报 **「尚未开通体检所需权限」**——读应用版本信息本身也需要授权
> （`application:application.app_version:readonly`）。点它给的链接一次把体检权限和机器人权限
> 全开通、发布版本，再回来「重新检查」，之后各项就都能自动判定了。

## 三、在 Lumi 桌面端配置

打开桌面端 **设置（齿轮）→ 渠道**，点「飞书」的 **编辑**：

| 字段 | 说明 |
|---|---|
| **App ID / App Secret** | 上一步拿到的凭证。支持 `${FEISHU_APP_ID}` 这样引用环境变量（避免明文）。 |
| **工具审批模式** | `AI 审批`（auto，默认，需审批的批次交分类器裁决）/ `特权放行`（privileged，自动放行）。两档下需人工确认的高危操作一律自动拒绝——飞书不弹审批卡。 |
| **群消息策略** | `@我才回`（默认，群里只响应 @机器人）/ `响应全部`。 |
| **可用成员（白名单）** | 默认「所有人」；选「指定成员」可只允许特定 open_id。 |
| **项目目录**（可选） | 飞书会话绑定的项目根，留空 = serve 进程当前目录。 |
| **妙记纪要** | 开关（默认关）。开启后，你录音或开会生成妙记时，Lumi 自动取回逐字稿并整理纪要，推送到与机器人的私聊。开关下方会逐项体检链路（lark-cli / 用户授权 / 妙记权限 / 事件订阅），哪一步没通就给出对应的命令或开放平台链接——详见下方「妙记纪要」一节。 |
| **每日记忆整理（Dream）** | 开关（默认关）。到点对有新消息的会话先沉淀记忆、再压缩历史，让常驻会话不无限膨胀。可设执行时间（建议低峰时段）与 Summary 最大并发（限流防接口 429）。 |

填好后：
1. 点 **测试连接** 验证凭证（成功会显示机器人名）。
2. 点 **保存并重连**，把开关打开 → 状态显示「已连接」。

配置存在 serve 所在机器的 `~/.lumi/lumi.json` 的 `channels` 分区（密钥仅本人可读）；保存即实时重连，无需重启。

> **连接状态**（渠道卡片上的文字）：未启用 / 已停止 / 连接中 / 已连接 / 连接失败（失败会显示
> 具体原因，如「缺少 app_secret」「未安装 lark-oapi」，文字转红）。掉线/重连期间会回落到「连接中」。

## 四、使用

- **私聊**：直接给机器人发消息。
- **群里**：把机器人拉进群，`@机器人` 发消息（除非把群策略设为「响应全部」）。
- **图片**：直接发图，或**回复一张图**再 @机器人问「这是什么」——它会读取被回复的那张图。
- **文件**：发 PDF / 文本 / 代码等，机器人会下载并用 `read` 读取（PDF 自动渲染成图）。
- **连发多条**：机器人在跑长任务时，你接着发的几条会排队，等它忙完**合并成一轮**一起理解
  （后发的可能是对前面的补充或更正），而不是丢掉。
- **斜杠命令**：消息以 `/命令名` 开头即触发（群里 `@机器人 /命令` 也行）：
  - `/help` —— 列出全部可用命令（带彩色标题卡片）；
  - `/stop` —— 停止正在执行的任务（包括它派生的后台任务），并清掉排队的消息；
  - `/clear` —— 清空本会话历史，重新开始；
  - `/compact` —— 压缩本会话历史（保留摘要），长期聊的群可手动瘦身；
  - `/dream-session` —— 立即整理本会话记忆（后台跑，完成会回音）；
  - 其余命令来自项目的技能目录（与 desktop 输入框的 `/` 补全同一套，如 `/commit`），
    命令后可以接补充说明，如 `/commit 只提交文档改动`。
  - 拼错或不存在的 `/xxx` 不报错，按普通消息交给模型理解。

每个会话（私聊 / 每个群）是**独立、持续**的对话，历史一直累积。

### 妙记纪要（自动出会议 / 录音纪要）

开启「妙记纪要」开关后：你**录音**或**开会（须开云录制）** → 飞书生成妙记 → Lumi 自动取回
**逐字稿**（带说话人与时间戳）→ 整理成纪要推到与机器人的私聊。实测录完约 20 秒送达。

首次启用要满足四个条件，配置弹窗里会逐项体检，没通过的直接给命令或链接：

| 检查项 | 没通过怎么办 |
|---|---|
| lark-cli 已安装 | 终端跑 `npm i -g @larksuite/cli` |
| 用户已授权 | 终端跑 `lark-cli auth login --recommend --scope "minutes:minutes.basic:read,minutes:minutes.transcript:export"` 扫码（读妙记内容必须**用户身份**，应用身份会被拒；login 只请求命令里指定的 scope，应用开通了也不会自动带上，所以必须显式 `--scope`） |
| 妙记权限已开通 | 点面板给出的开放平台链接，在「**用户身份权限**」tab 下开通 `minutes:minutes.basic:read` 与 `minutes:minutes.transcript:export`（应用身份不生效）并**发布版本**，然后重跑上面的 login 命令使新权限进 token |
| 事件订阅生效 | 开放平台「事件与回调」添加 `minutes.minute.generated_v1`（添加后需发布版本）。Lumi 每次启动会自动重建订阅 |

> ⚠️ 授权有效期 **7 天**（refresh_token），超期需重新 `lark-cli auth login`。
> 另外**别用 `lark-cli event consume` 长期占用**——它退出时会主动取消订阅，导致妙记功能静默失效；
> 真发生了，重开一次配置弹窗即可自动修复（体检第四项会重建订阅）。

## 五、常见问题

- **机器人不回**：①状态灯是否「已连接」（红光看原因）；②飞书后台事件订阅是否选了**长连接**模式
  并订阅了 `im.message.receive_v1`；③群里是否 @ 了机器人（默认 @我才回）；④白名单是否把你排除了。
- **日志报 `connect failed, requires python-socks`**：环境有 SOCKS 代理但缺 `python-socks`，
  重新 `uv sync`。
- **看不懂图**：确认你在「设置 → 模型」选的是**支持视觉**的多模态模型。
- **多机/远程**：飞书 channel 跑在 serve 所在的机器，配置和凭证也在那台机器；desktop 经 WS 远程
  编辑即可。
