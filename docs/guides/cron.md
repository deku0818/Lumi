# 定时任务使用指南

Lumi 内置定时任务系统，让 Agent 按照设定的时间规则自动执行任务并投递结果。

---

## 快速上手

在对话中直接告诉 Lumi 你想定时做什么：

```
用户：每天早上 9 点帮我总结一下今天的待办事项
Agent：✅ 任务已创建：每日待办总结（ID: a1b2c3d4e5f6，调度: cron 0 9 * * *）
```

也可以手动指定调度规则：

```
用户：每 30 分钟检查一下服务器状态
用户：2025-03-10T14:00:00 提醒我开会
```

---

## 调度规则

支持四种格式，`cron` 工具的 `schedule` 参数会自动识别：

| 类型 | 格式 | 示例 | 说明 |
|------|------|------|------|
| 相对时间 | `+<数字><单位>` | `+10m`、`+2h`、`+1d` | 从现在起，转换为一次性执行 |
| 固定间隔 | `<数字><单位>` | `30s`、`5m`、`2h`、`1d` | 单位：s(秒) m(分) h(时) d(天) |
| 一次性 | ISO 8601 | `2025-03-10T14:00:00` | 执行后自动删除 |
| cron 表达式 | 5 字段 | `*/5 * * * *`、`0 9 * * *` | 分 时 日 月 周 |

**cron 表达式常用示例：**

| 表达式 | 含义 |
|--------|------|
| `*/5 * * * *` | 每 5 分钟 |
| `0 9 * * *` | 每天 9:00 |
| `0 9 * * 1-5` | 工作日 9:00 |
| `0 */2 * * *` | 每 2 小时 |
| `0 0 1 * *` | 每月 1 号 0:00 |

---

## 操作一览

`cron` 工具支持 7 种操作：

| 操作 | 必填参数 | 说明 |
|------|----------|------|
| `create` | name, schedule, prompt | 创建任务 |
| `list` | — | 列出所有任务 |
| `update` | job_id + 可选 name/schedule/prompt | 修改任务 |
| `delete` | job_id | 删除任务 |
| `run` | job_id | 立即执行一次（不影响调度计划） |
| `pause` | job_id | 暂停/恢复切换 |
| `runs` | job_id | 查看最近执行记录（默认 20 条） |

---

## TUI 管理

- `/cron` — 查看和管理定时任务（支持删除）
- `/cron-notify` — 查看定时任务通知记录

执行中的任务会在输入栏右侧显示动画指示器（如 `▰▱▱ 每日总结 执行中`）。

---

## 结果投递

任务执行结果会广播到所有已注册的投递通道：

| 通道 | 说明 |
|------|------|
| TUI | 持久化到通知存储，通过 `/cron-notify` 查看 |
| API | 通过 SSE 推送到 `GET /api/cron/events` |

---

## 错过任务补偿

Lumi 启动时会检查离线期间错过的任务并自动补偿执行一次。
