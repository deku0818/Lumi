# 记忆与项目说明使用指南

Lumi 会跨会话记住关于你和项目的关键信息，并能读取项目根的 `LUMI.md` 作为「这个项目的约定」。
你基本不用管它——对话里告诉它该记的，它自己存；下次对话自动带上。

---

## 持久记忆：让 Lumi 记住

直接在对话里说，它就会存下来：

```
你：以后回复简短点，别在结尾总结
Lumi：好的（存为 feedback 记忆）

你：我主要写后端，React 是新手
Lumi：明白（存为 user 记忆，以后前端的解释会给你多铺垫）
```

也可以显式要求：「记住……」「忘掉关于……的记忆」。

**它会记什么**（四类）：
- **你是谁**（角色、专长、偏好）
- **怎么协作**（你纠正过 / 确认过的工作方式）
- **项目背景**（进行中的事、事故、决策等代码里看不出的上下文）
- **外部指针**（bug 在哪个看板、文档在哪）

**它不会记什么**：代码结构 / git 历史 / 调试修复配方 / 临时状态——这些读代码或 `git log` 即可得知。
即便你让它存这类，它会反问「哪里是不显然的」，只留那部分。

### 存在哪

`~/.lumi/memory/projects/<你的项目路径>/`，按项目隔离、跨会话保留。里面：
- `MEMORY.md` —— 索引（每行一条，开会话时整体进上下文）
- 各 `*.md` —— 单条记忆正文（需要时 Lumi 自己 grep/读）

> 记忆可能过时。Lumi 在据某条记忆推荐前会先验证（提到的文件/函数是否还在），与现状冲突时以现状为准并更新记忆。

### 关于审批

写记忆文件**不会弹审批**（记忆的价值就是不打断对话）。但你配置的 DENY 规则、readonly/plan 执行模式仍然
拦得住它——免审批只免掉「本该问你一下」那一步。

---

## `LUMI.md`：项目说明

在项目根放一个 `LUMI.md`，写这个项目的约定、对 Lumi 的指示，它会在每次会话开头读进来：

```markdown
# 本项目约定
- 依赖用 uv 管理，别直接改 pyproject
- 测试命令：uv run pytest
- 提交信息用中文
```

- 主 agent 和子 agent 都会读到它。
- 它和「持久记忆」是两回事：`LUMI.md` 是你手写的项目规矩，记忆是 Lumi 自己积累的。
- **`LUMI.md` 默认不进 git**（已在 `.gitignore`）：内容随项目 / 开发者而异，本地维护。想团队共享就把它从
  `.gitignore` 去掉自行提交。

---

## 关掉记忆

记忆默认只对**面向用户的桌面/对话会话**开启；子 agent、定时任务（cron）、workflow 天然不带记忆。
若要整体不用，删掉或清空 `~/.lumi/memory/projects/<项目>/` 即可（`LUMI.md` 注入不受影响，单独删文件即停）。
