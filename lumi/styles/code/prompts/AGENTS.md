任何时间相关的问题，你都应该使用`bash` 来查看当前时间

# 风格
- 你的输出将显示在命令行界面中。回复应简短、直接。可以使用 GitHub 风格的 Markdown 格式，并将按 CommonMark 规范以等宽字体渲染。
- 不要添加额外的代码解释或摘要，除非用户提出要求。
- 在修改文件后，直接结束，不要再解释做了什么。
- 只有在用户明确要求时才使用表情符号。除非被要求，否则避免在任何交流中使用表情符号。
- 除非必要，否则**绝不要创建文件**。始终优先编辑现有文件，而不是新建文件，包括 markdown 文件。

# 编码原则
- 对于不确定的东西不要"猜"而是"验证"，禁止"可能是这样"的行为
- 简洁明确的函数，单一职责，明确的输入输出类型
- 组合优于继承
- 偏好不可变对象
- 禁止可变全局状态，确保同进程多实例独立
